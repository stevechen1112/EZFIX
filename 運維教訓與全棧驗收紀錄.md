## 四、歷史問題與教訓（今天與之前）

> **核心教訓**：`ContentFlow_整合實作記錄.md` 第六節驗收清單全部 `[x]`，但多項從未在 **https://ezfix.com.tw** 做過端到端驗證。

### 4.1 「文件勾完成」但實際沒做的事

| 問題 | 表面狀態 | 實際狀況 | 教訓 |
|---|---|---|---|
| GA4 已安裝 | 文件 ✅ | `layout.tsx` 長期無 gtag；`.env` 無衡量 ID | **必須 curl 正式站 HTML + GA 即時報表雙重驗證** |
| 正式站驗收 | 文件 ✅ | 未對 live URL 做 checklist | 驗收必須附 **curl / 瀏覽器 / 後台截圖** 證據 |
| ContentFlow 端到端 | API 程式 ✅ | 平台未部署；Secret 未輪替 | 「接收端就緒 ≠ 整合完成」 |

### 4.2 DNS 指向錯誤伺服器（6 月重大事故）

| 時間點 | 狀況 |
|---|---|
| 問題 | `ezfix.com.tw` 指向 `165.154.226.93`（Caddy 反向代理），應用在 `104.64.142.40`（Linode） |
| 現象 | Linode 直連 200，公開網域 404；圖片全挂 |
| 根因 | 部署修在 Linode，DNS 仍走舊機 |
| 教訓 | **每次修復後必查 `nslookup ezfix.com.tw` 與公開 URL**，不能只看 SSH 本機 curl |

### 4.3 圖片未隨部署上線

| 問題 | 根因 | 修復 |
|---|---|---|
| 前台無圖 | `public/uploads/*` 在 `.gitignore`；deploy 只建空目錄 | `sync-uploads.ps1` + `bind-upload-images.ts` |
| Caddy 攔截 `/uploads/` | 舊代理不轉發靜態檔 | 改 `/media/` 由 Next.js 提供 + 301 相容舊路徑 |
| DB 圖片路徑為空 | seed 後未 bind | `bind-upload-images.ts` |

**教訓**：靜態資產需有 **明確同步流程**（腳本或 object storage），不能假設 git deploy 會帶圖。

### 4.4 瀏覽器快取造成「我這邊壞、你那邊好」

| 現象 | 原因 |
|---|---|
| 無痕正常、一般視窗 404 / 無圖 | DNS 切換前快取的舊 HTML、舊 `/_next/static/*` hash |
| Console 出現舊 CSS 404 | 舊 build 的 chunk 已不存在 |

**教訓**：對使用者報障時，先請 **清除網站資料** 再判斷伺服器問題。

### 4.5 GA 與 GSC 數據對不起來

| 現象 | 解釋 |
|---|---|
| GSC 有 162 點擊，GA 6/3 後歸零 | 追蹤碼未安裝 → 有搜尋曝光點進來，但 GA 收不到 |
| 修 GA 後仍看不到搜尋字詞 | GA↔GSC 未連結（已於 2026-06-28 修復） |

**教訓**：GSC 點擊 ≠ GA 工作階段；兩邊都要獨立驗證，且需 **產品連結** 才能在 GA 看自然搜尋字詞。

### 4.6 部署與版控脫節

| 問題 | 說明 |
|---|---|
| 伺服器 `git log` 僅 Initial commit | 實際程式可能靠 tar/scp 覆蓋，非 git pull |
| 本機 20+ 檔案未 commit | 含 GA、media、deploy 改動 |
| CI 無 post-deploy 驗證 | push main 後不知道 GA/圖片/API 是否仍正常 |

**教訓**：統一 **單一部署來源**（git pull 或 Actions artifact），並加 **部署後 smoke test**。

### 4.7 健康檢查與測試資料污染

| 問題 | 說明 |
|---|---|
| 用 publish API 測試 | 留下 `cf-health-check-delete-me` draft |
| 無 health endpoint | 無法一鍵確認「站還活著 + GA env 有設 + DB 可連」 |

**教訓**：測試用 slug 加 `-delete-me` 後綴，測完 **腳本自動清理**；或改用 `GET /api/health`。

### 4.8 `.env` 人為錯誤

| 問題 | 說明 |
|---|---|
| 兩行 env 黏在一起 | 導致 GA ID 未生效 |
| Secret 用預設弱密碼 | 文件與程式 fallback 都是可猜的字串 |

**教訓**：部署後執行 `grep` 確認每行一個變數；Secret **禁止** commit 到 git。

## 五、標準驗收腳本（每次部署後必跑）

在可連正式站的環境執行：

```powershell
# 1. 基本可用性
@('/', '/blog', '/robots.txt', '/sitemap.xml', '/admin/login') | ForEach-Object {
  $u = "https://ezfix.com.tw$_"
  try { "$u -> $((Invoke-WebRequest $u -UseBasicParsing -TimeoutSec 15).StatusCode)" }
  catch { "$u -> FAIL" }
}

# 2. GA 追蹤碼
$html = (Invoke-WebRequest 'https://ezfix.com.tw/' -UseBasicParsing).Content
if ($html -match 'G-CXF5F2DJK1') { 'GA: OK' } else { 'GA: MISSING' }

# 3. 圖片
@('/media/hero/hero-1.jpg', '/media/services/service-1.jpeg') | ForEach-Object {
  try { "https://ezfix.com.tw$_ -> $((Invoke-WebRequest "https://ezfix.com.tw$_" -Method Head -UseBasicParsing).StatusCode)" }
  catch { "https://ezfix.com.tw$_ -> FAIL" }
}

# 4. ContentFlow API（應回 401）
try { Invoke-WebRequest 'https://ezfix.com.tw/api/contentflow/publish' -Method POST -UseBasicParsing }
catch { "ContentFlow auth: $($_.Exception.Response.StatusCode.value__)" }

# 5. DNS
nslookup ezfix.com.tw
```

**人工確認（後台）**：
- GA4 即時報表有使用者
- GA 管理 → Search Console 連結存在
- GSC → Sitemap 上次讀取成功
