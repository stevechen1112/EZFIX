## 三、發現的問題與根因分析

### 問題 1：索引覆蓋率極低（4.1%）— 🔴 嚴重

**根因**：
1. **全站缺乏 canonical URL**：`layout.tsx` 的 `metadata` 未設 `alternates.canonical`，文章頁 `generateMetadata` 也未設。Google 爬蟲無法判斷標準網頁，導致 2 頁被判定為「重複網頁，未選取標準」。
2. **缺乏 `metadataBase`**：Next.js 的 OpenGraph URL 和 canonical 無法正確解析為絕對 URL。
3. **`force-dynamic` 全面使用**：所有頁面每次請求都 SSR，爬蟲可能拿到不穩定的 HTML。
4. **網站太新**（3.8 個月）：Google 對新站有觀察期，39 頁「已檢索但未建立索引」部分與此有關。
5. **內容品質**：blog 僅 4 篇已發布文章，部分可能被判定為薄內容。

**影響**：70 頁無法出現在 Google 搜尋結果中，嚴重限制自然流量。

### 問題 2：23 個 404 頁面 — 🟠 高

**根因**：網站從 WordPress 遷移至 Next.js，舊 WP URL（`/wp-admin/`、`/wp-content/uploads/`、`/category/`、`/tag/`、`/feed`、`?p=ID` 等）仍被 Google 索引，但新站不存在這些路由。

**影響**：爬蟲預算被浪費在 404 頁面，且 GSC 報表持續顯示錯誤。

### 問題 3：OpenGraph 與 Twitter Card 不完整 — 🟡 中

**根因**：`layout.tsx` 的 `openGraph` 僅設 `type`、`locale`、`siteName`，缺 `url`、`title`、`description`、`images`。`twitter` 完全未設定。文章頁 `generateMetadata` 也無 OG/Twitter 欄位。

**影響**：社群分享（FB、LINE、Twitter/X）時無預覽圖片、標題不完整，降低點擊率。

### 問題 4：BreadcrumbList 結構化資料缺失 — 🟡 中

**根因**：首頁有 `LocalBusiness` schema、文章頁有 `Article` schema，但全站無 `BreadcrumbList` schema。GSC「導覽標記」報表顯示 0 有效。

**影響**：搜尋結果中無法顯示麵包屑導覽，降低 SERP 點擊率。GSC 強化項目報表無資料。

### 問題 5：Sitemap 缺少 changeFrequency 和 priority — 🟡 低

**根因**：`sitemap.ts` 只提供 `url` 和 `lastModified`，缺 `changeFrequency` 和 `priority` 欄位。blog 列表頁的 `lastModified` 用 `new Date()`（每次請求都變）。

**影響**：Google 無法判斷頁面更新頻率，可能降低檢索效率。

### 問題 6：Article Schema dateModified 不正確 — 🟡 低

**根因**：`buildArticleSchema` 中 `dateModified` 使用 `publishedAt`（發布時間）而非 `updatedAt`（更新時間）。

**影響**：Google 無法正確識別文章最後更新時間，影響時效性內容的排名。

## 四、修復內容詳解

### 修復 1：`app/layout.tsx` — 全站 metadata 強化

**修改前**：
```typescript
export const metadata: Metadata = {
  title: { default: "...", template: "%s | ..." },
  description: "...",
  openGraph: { type: "website", locale: "zh_TW", siteName: "..." },
};
```

**修改後新增**：
- `metadataBase: new URL(siteUrl)` — 讓所有相對 URL（OG image 等）解析為絕對 URL
- `robots` — 全站 `index: true, follow: true` + googleBot 進階指令（`max-image-preview: large`）
- `alternates.canonical` — 首頁標準 URL
- `openGraph.url` / `openGraph.title` / `openGraph.description` / `openGraph.images` — 完整 OG
- `twitter.card: "summary_large_image"` + title / description / images — Twitter 分享卡片

### 修復 2：`app/blog/[slug]/page.tsx` — 文章頁 SEO 全面強化

**修改前**：
```typescript
export async function generateMetadata({ params }) {
  return {
    title: post.metaTitle || `${post.title} | 恆惠修理紗窗`,
    description: post.metaDescription || post.excerpt || post.title,
  };
}
```

**修改後新增**：
- `alternates.canonical` — 每篇文章的絕對 canonical URL
- `openGraph.type: "article"` + `publishedTime` / `modifiedTime` / `authors` / `images`
- `twitter.card: "summary_large_image"` + images
- `robots: { index: true, follow: true }`
- **`buildBreadcrumbSchema()`** — 新函數，生成 `BreadcrumbList` JSON-LD
- **視覺麵包屑導覽** — `<nav aria-label="麵包屑">` + `<ol>` 結構
- `buildArticleSchema()` 改用 `updatedAt` 作為 `dateModified`
- `buildArticleSchema()` 新增 `url` 欄位

### 修復 3：`app/blog/page.tsx` — blog 列表頁 metadata

**修改前**：
```typescript
export const metadata = {
  title: "專業分享 - 紗窗維修知識懶人包 | 恆惠修理紗窗",
  description: "...",
};
```

**修改後新增**：
- 型別改為 `Metadata`（含完整型別檢查）
- `alternates.canonical` — `/blog` 的標準 URL
- `openGraph` — 完整 OG（type、url、title、description、images）
- `twitter` — summary_large_image 卡片
- `robots: { index: true, follow: true }`

### 修復 4：`app/sitemap.ts` — sitemap 強化

**修改前**：
```typescript
return [
  { url: buildPublicUrl(baseUrl, "/"), lastModified: new Date() },
  { url: buildPublicUrl(baseUrl, "/blog"), lastModified: new Date() },
  ...posts.map((post) => ({
    url: buildPublicUrl(baseUrl, `/blog/${post.slug}`),
    lastModified: post.updatedAt,
  })),
];
```

**修改後新增**：
- `changeFrequency` — 首頁 `"weekly"`、blog 列表 `"weekly"`、文章 `"monthly"`
- `priority` — 首頁 `1.0`、blog 列表 `0.8`、文章 `0.6`
- `images` — 文章封面圖加入 sitemap（image sitemap）
- blog 列表 `lastModified` 改用最新文章的 `updatedAt`（而非 `new Date()`）
- 文章查詢加入 `coverImage` 欄位和 `orderBy: { publishedAt: "desc" }`

### 修復 5：`next.config.mjs` — 舊 WordPress URL 301 轉址

**新增 11 條 redirect 規則**：

| 舊 WP URL | 轉址目標 | 類型 |
|-----------|---------|------|
| `/index.php` | `/` | 301 |
| `/wp-content/uploads/:path*` | `/media/:path*` | 301 |
| `/wp-admin/:path*` | `/` | 301 |
| `/wp-login.php` | `/` | 301 |
| `/feed` | `/blog` | 301 |
| `/xmlrpc.php` | `/` | 301 |
| `/category/:path*` | `/blog` | 301 |
| `/tag/:path*` | `/blog` | 301 |
| `/author/:path*` | `/blog` | 301 |
| `/page/:path*` | `/blog` | 301 |
| `/?p=*` (query matching) | `/blog` | 301 |

**影響**：消除 GSC 回報的 23 個 404 頁面，Google 重新檢索後會將這些 URL 從索引中移除或更新。
